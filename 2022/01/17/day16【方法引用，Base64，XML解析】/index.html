第一章 : 方法引用 [ 了解 ]

能够知道方法引用的四种方式就可以了。

1）方法引用概述

  • 当使用Lambda实现一个逻辑时，如果这个逻辑已经在某个类中存在相同逻辑的方法 ，可以直接引用此功能而不需要写Lambda 。

    我们可以理解为Lambda用来简化匿名内部类 , 方法引用是用来简化Lambda。

2）方法引用的分类

  • 方法引用的符号 , 是双冒号 **::**
种类 语法格式
1：静态方法引用 类名 :: 静态方法名
2：构造方法引用 类名 :: new
3：类的任意对象的方法引用 类名 :: 实例方法名
4：特定对象的方法引用 对象 :: 实例方法名

1 Base64概述

  • 1 ：要求理解Base64编码原理

  • 2 ：Base64 编码是一种常用的字符编码，在很多地方都会用到，他核心作用是保证传输数据的正确性

  • 3：有些系统只能使用ASCII字符(本文字符) , Base64就是用来将非ASCII字符的数据转换成ASCII字符的一种方法

  • 4：而且base64特别适合在后面学习到的 http , mime 协议下快速传输数据。

  • 5：base64不是安全领域下的加密解密算法。能起到安全作用的效果很差，而且很容易破解。（只是对数据的编解码 , 不是加密解密算法）

第三章 XPath

第一 ： XPath解析XML介绍

  • 能够导入xpath所需jar包，能够熟悉xpath相关api

  • 前言 : Dom4J和XPath的区别

    • Dom4J : 层级递进方式获取元素
    • Xpath : 通过路径筛选想要的元素##(灵活多了)
    • ps: 都属于DOM解析思想
1) xPath介绍 :

  • XPath 使用路径表达式来选取XML/HTML 文档中的元素节点或属性节点 , 节点是通过沿着路径 (path) 来选取的。

  • XPath在解析XML/HTML文档方面提供了一种路径解析思想。

2) xpath解析步骤 :
3) API :
  • Node接口中存在以下方法：
方法 作用
List selectNodes(“路径”) 获取符合路径的元素集合
Node selectSingleNode(“路径”) 获取符合路径的唯一元素
  • 我们熟知的Document，Element等都是Node的子类型，因此Document，Element对象也能使用上述的方法。
4) XPath语法（解析html，结合正则表达式用来爬虫，方便得狠！）

  • XPath路径，就是用于选取XML文档中节点的表达式字符串。

  • 获取XML文档节点元素一共有如下4种XPath语法方式：

    • 1：绝对路径

      例如 : /元素/子元素/子子元素…

    • 2：相对路径

      • 例如 : 子元素/子子元素.. 或者 ./子元素/子子元素..

    • 3：全文搜索路径

      • 例如 : **//**子元素1//子元素2 (所有子元素1标签下的所有 子元素2标签)
      • 注意：只要2 在1 下就可以了，无论他们是父子关系还是孙子关系等等。

    • 4：谓语（条件筛选)

      • 例如 : //元素1[@attr] （必须有某属性）
      • 又例如 : //元素1[@attr=”it001”] (属性的值为具体值)
