1.5 jsp执行流程 ★

​ 第一次访问jsp(index.jsp)页面时,服务器接收请求,由jspservlet来处理该请求.

当浏览器访问 *.jsp 时,统一由JspServlet处理这类请求,

找到对应的jsp页面,转成java类(就是Servlet)

然后调用转完后的servlet类中的_jspService方法,在该方法中将jsp页面中的html代码以响应流的形式返回给浏览器展示.

jsp本质上就是servlet,就是一个运行在服务器上的java类.
只不过这个java类,事先不存在,在服务器的运行过程中动态的生成的.
生成后的java类存放在tomcat服务器的work目录下.

用自己的话总结就是：

前面学的servlet方式是在servlet实现类（java文件）中实现将html代码返回给浏览器。而有了jsp之后，可以根据规则直接写jsp文件，里面包含了html代码，浏览器可以直接访问jsp文件，当被访问到时，tomcat将jsp转为 servlet实现类，再将html以流的形式返回给浏览器。对比直接用servlet方法返回html给浏览器，jsp方法更繁琐，而且效率也略低一点，但是由于具有jsp约束，jsp具有更多的功能，好处。接下来讲解jsp特有内容来体现这种动态技术的好处！

在 JSP 文件中，HTML 代码与 Java 代码共同存在。其中，HTML 代码用于实现网页中静态内容的显示，Java 代码用于实现网页中动态内容的显示。为了与传统 HTML 有所区别，JSP 文件的扩展名为 jsp。
